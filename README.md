# 🎧 REI Discord Bot – бот для автоматического создания и управления приватными комнатами

Бот сам создаёт временные приватные голосовые комнаты, когда ты заходишь в канал-триггер. Комнаты удаляются, как только последний участник их покидает. Можно настроить роли, которые имеют доступ во все созданные комнаты (без приглашения), или владелец комнаты через интерфейс Discord может настроить доступ для конкретных людей.

---

## 📖 Для пользователей бота

### 🔧 Основные возможности

- Когда ты заходишь в заранее назначенный голосовой канал (канал-триггер), бот создаёт новую приватную комнату с именем, равным твоему нику.
- Созданная комната видна всем, но зайти в неё могут только:
    - владелец комнаты (тот, кто её создал);
    - люди, которых владелец добавит через интерфейс Discord;
    - участники с ролями, добавленными в список исключений (опционально);
    - администраторы сервера (через ручную настройку прав).
- Когда комната пустеет, бот автоматически удаляет её через 5 секунд.
- После перезапуска бот восстанавливает контроль над ранее созданными комнатами и удаляет пустые.

### 🛠️ Команды

| Команда                          | Описание                                                                                               |
|----------------------------------|--------------------------------------------------------------------------------------------------------|
| `/set_default_channel #канал`    | Устанавливает голосовой канал, заходя в который ты будешь получать новую приватную комнату.            |
| `/get_default_channel`           | Показывает текущий канал-триггер.                                                                      |
| `/add_exception_role @роль`      | Добавляет роль в список исключений – участники с этой ролью смогут заходить в любые приватные комнаты. |
| `/remove_exception_role @роль`   | Удаляет роль из списка исключений.                                                                     |
| `/get_exception_roles`           | Показывает список ролей-исключений.                                                                    |
| `/update_active_channels_access` | Применяет текущий список ролей-исключений ко всем уже созданным комнатам (обновляет права доступа).    |

> Все команды пока что требуют прав **администратора** сервера. В будущем некоторые из них станут доступны всем.

### 📌 Пример использования

1. Создай на сервере голосовой канал, например, «➕ Создать комнату» (можно использовать любой другой).
2. Напиши команду: `/set_default_channel` и выбери нужный голосовой канал из списка, затем отправь команду.
3. Теперь любой участник, зайдя в этот канал, автоматически получит свою приватную комнату с его ником.
4. Если ты хочешь, чтобы твои друзья могли заходить в комнаты друг друга, добавь человека или роль через настройки канала в Discord.
5. Если ты хочешь, чтобы все пользователи с определённой ролью (например, «Модератор») могли заходить во все созданные приватные каналы, добавь роль в исключения командой: `/add_exception_role` и выбери нужную роль, затем отправь команду.
6. Чтобы применить правила к уже существующим комнатам, выполни `/update_active_channels_access`.

## 🧑‍💻 Для разработчиков / контрибьюторов

### 🚀 Развёртывание собственного экземпляра бота

#### Требования

- Node.js **18+** (рекомендуется 20 LTS)
- npm или yarn
- [Discord Application](https://discord.com/developers/applications) с ботом и токеном

#### Установка и запуск

```bash
# Клонируй репозиторий
git clone https://github.com/Z1XWP/rei-discord-bot.git
cd rei-discord-bot

# Установи зависимости
npm install

# Создай файл .env по примеру (см. ниже)
cp .env.example .env

# Скомпилируй TypeScript
npm run build

# Запусти бота (в production через PM2)
pm2 start dist/index.js --name rei-discord-bot
pm2 save
pm2 startup

Переменные окружения (.env)

DISCORD_TOKEN=твой_токен_бота
CLIENT_ID=ID_приложения
GUILD_ID=ID_сервера (для регистрации команд)
CLIENT_ID можно скопировать из General Information приложения в Discord Developer Portal. GUILD_ID – из контекстного меню сервера (режим разработчика).

Файлы конфигурации (создаются автоматически). При переезде на БД часть станет неактуальной
config.json – хранит ID канала-триггера.

rooms.json – список созданных комнат (восстанавливается после перезапуска).

exception-roles.json – список ролей-исключений.



🧪 Разработка и тестирование
bash

npm run dev

🔌 Расширение функциональности
Добавление новой команды:
1. В commands.ts добавь SlashCommandBuilder в массив commandsData.
2. Напиши обработчик (например, async function handleNewCommand(bot, interaction) {...}).
3. Зарегистрируй обработчик в commandHandlers Map.
